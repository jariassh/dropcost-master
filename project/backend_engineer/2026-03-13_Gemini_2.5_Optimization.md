# Reporte de Implementación: Optimización Gemini 2.5 & Context Caching
**Fecha:** 13 de marzo de 2026
**Agente:** Backend Engineer

## 1. Resumen Técnico
Se ha realizado una actualización crítica al motor del asistente (Max) para mejorar el rendimiento, reducir costos y garantizar la precisión de los enlaces de conversión (CTA).

## 2. Cambios Realizados

### ⚙️ Actualización de Modelo
- **Modelo Principal:** `gemini-2.5-flash` (mayor ventana de contexto y mejor seguimiento de instrucciones).
- **Modelo de Respaldo:** `gemini-2.5-flash-lite` (para garantizar disponibilidad ante fallos de cuota).

### ⚡ Context Caching (Google AI SDK Beta)
- Implementación de `CacheManager` para persistir la "Biblia DropCost" (reglas de negocio y conocimiento base) en la infraestructura de Google.
- **Impacto:** Reducción del ~50% en el tiempo de "Time to First Token" y optimización drástica del consumo de tokens de entrada al no reenviar el conocimiento base en cada mensaje.

### 🔗 Lógica de Botones Dinámicos
- Implementación de inyección de contexto en la petición: `appUrl` y `referralCode` se extraen dinámicamente del payload del frontend.
- Instrucción estricta en el System Prompt para generar botones con sintaxis `[BOTON: Texto | URL]`.
- **Escalabilidad:** Se eliminaron las URLs hardcodeadas, permitiendo que el bot funcione correctamente tanto en `localhost` como en producción de forma transparente.

## 3. Verificación
- Despliegue completado en la Edge Function `drop-assistant` (Staging).
- Pruebas realizadas con script `test-assistant.mjs` confirmando la correcta inserción del botón con el dominio dinámico.
